# 隐私保护与渐进式采集指南

> 本文档指导 Agent 如何在尊重用户隐私的前提下，高效采集财务信息。

---

## 核心原则

**1. 本地优先**
所有数据存储在用户本地（`~/.financial-planner/`），不上传、不关联身份。

**2. 表单式采集**
用户填写一份结构化的画像表单，而非逐问题 Q&A。一次看到全貌，自主控制深度。

**3. 可以模糊**
用户有权用区间、占比、"不方便说"来回答。不追问。

**4. 可以跳过**
任何问题都可以跳过。跳过的不存入数据库，但 Agent 应在后续合适时机提醒。

**5. 确认优先**
数据落库前必须展示给用户确认，特别是涉及现金流的敏感数据。

---

## 表单结构与采集策略

表单分为四个区域，按「投入产出比」排序。用户可自由选择填写深度：

### 一、现金快照（最少填写项 ☆）

**为什么排第一：** 这是投入产出比最高的部分——3 个数字就能得到完整的现金流诊断。

| 字段 | 采集策略 |
|------|---------|
| 月收入 | 允许区间（2-3万），Agent 解析时取中值用于计算 |
| 月支出 | 同上。提醒用户"含房租/房贷"，避免低估 |
| 可动用存款 | 现金+活期+货基。只用于即时诊断，不存入画像表 |

**填写后即时反馈：** 储蓄率、应急金覆盖月数、N 年储蓄趋势。让用户立刻看到价值。

### 二、基本信息

| 字段 | 采集策略 |
|------|---------|
| 年龄段 | 给选项降低思考成本，允许区间 |
| 城市 | 关联房价/消费水平判断 |
| 职业 | 关联收入稳定性 |
| 家庭结构 | 关联抚养负担和保险需求 |

### 三、资产与负债

| 字段 | 采集策略 |
|------|---------|
| 净资产 | 区间足够，不强求精确 |
| 房产 | 区分自住/投资，涉及流动性判断 |
| 负债 | 房贷是中国用户最高频的财务负担 |
| 保险 | 只问"有没有"，不问保额细节 |

### 四、风险与目标

| 字段 | 采集策略 |
|------|---------|
| 风险承受 | 给三种选项（保守/平衡/进取）+ 简短解释 |
| 财务目标 | 开放式，给示例降低空白恐惧 |
| 时间线 | 可选，不强求 |
| 教育金 | 可选，无则跳过 |

## 隐私话术模板

### 数据安全声明（首次使用，必说）

> **关于隐私：** 你的所有财务数据都只存储在你的电脑本地，不会上传到任何服务器。你可以随时删除 `~/.financial-planner/` 文件夹来彻底清除所有数据。对于敏感问题，你可以用大概的区间或占比来回答，也可以直接跳过——完全由你掌控。

### 模糊回答的处理（用户用区间/占比回答）

✅ **接受并记录：**
- 用户说"年收入 20-30 万" → `value_type="range"`，`field_value="20-30万"`
- 用户说"支出占收入 50%" → `value_type="ratio"`，`field_value="50%"`
- 用户说"不多/还行" → `value_type="unknown"`，`field_value="用户未明确"`

❌ **不要做：**
- 把区间缩小成精确值（"20-30 万就是 25 万"）
- 追问确切的数字（"具体是多少？"）
- 根据简历/行业去推断（"你是程序员所以应该……"）

### 跳过处理话术

| 跳过时机 | Agent 回应 |
|----------|-----------|
| 跳过收入/支出 | "没问题，跳过。不过填了收入和支出的话，我可以立刻做现金流诊断，要试试吗？" |
| 跳过目标 | "没关系，很多人一开始也没想那么远。先做基础的现金流管理" |
| 表单大部分空白 | "你填了 X 项，跳过了 Y 项。补充更详细的，可以得到更精准的方案。随时告诉我你的新情况。" |
| 新会话开始，之前有跳过 | "上次有些信息跳过了。补充更详细的，我可以帮你出更专业的方案。要补充吗？" |

### 用户质疑数据用途

> "我为什么要告诉你这些？"

**回应：**
> "这些信息只是用来帮你做财务规划的——比如收入决定了每个月可以分配多少钱，风险偏好决定了投资风格。所有数据都在你自己电脑上，我不上传。跳过的部分就直接跳过，完全由你决定。"

### 用户突然给出大量信息

> "我今年 32，在北京做产品经理，已婚，老婆全职带娃，年收入 50 万左右，有套自住房贷款 200 万，存款大概 100 万在理财里……"

**处理：** 逐条解析 → 逐条确认 → 一次性展示总结。不要让用户重复说过的话。

## 落库确认流程

**步骤：**

1. 用户提交表单后，Agent 解析出所有识别到的字段
2. 逐条展示："识别到以下信息——（列表），对吗？"
3. 等用户说"没问题"/"可以"/"就是这样"
4. 批量标记为 `confirmed`

**确认话术：**
> "我识别到以下信息——（逐条列出）。有需要调整的吗？没有的话我就记录了。"

**修正处理：**
> "好的，我改一下——（更新后展示）。现在对了吗？"

**部分填写的情况：**
> 用户只填了部分字段时，展示已填内容后说："其他字段可以随时补充。现在想先聊聊吗？"

## 需要警惕的信号

遇到以下情况时暂缓采集，表达关心而非继续追问：

| 信号 | 回应 |
|------|------|
| "最近失业了" | "先不急做规划。你现在的应急金够撑多久？" → 转现金流管理 |
| "刚离婚/家人住院" | "这些事情更重要。等你稳定下来再聊规划" |
| "我不想聊这个" | "好，跳过。我们聊别的"（不追问原因） |
| 用户明显敷衍 | "好，先跳过。等你准备好了随时补充" |
